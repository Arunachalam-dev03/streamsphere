StreamSphere
Platform Documentation

A full-stack, production-grade video sharing platform built as a modern YouTube alternative. Features video upload & transcoding, Shorts, live streaming, creator tools, community engagement, admin moderation, and automated SEO — all containerized with Docker.

🏗️ Architecture & Tech Stack

Frontend (Client)

Next.js 14 React 18 Zustand Tailwind CSS HLS.js React Hot Toast
  • Routing: Next.js App Router with dynamic [handle], [id] segments for channels, videos, shorts
  • State: Zustand stores — useAuthStore (user, tokens), useUIStore (sidebar, miniPlayer)
  • Styling: Tailwind CSS with custom design tokens, dark/light theme via next-themes
  • Video: HLS.js for adaptive bitrate streaming, custom VideoPlayer with quality/speed/captions controls
  • Icons: React Icons (HeroIcons, Material Design, Simple Icons)

Backend (Server)

Node.js Express.js Prisma ORM Zod FFmpeg Node-Media-Server
  • API: Express REST API with JWT authentication (access + refresh tokens)
  • Validation: Zod schemas for all request payloads (query, body, params)
  • Transcoding: FFmpeg background workers for HLS segmentation, thumbnail generation
  • Live: Node-Media-Server (NMS) for RTMP ingest → HLS output on port 1935
  • Scheduling: Cron service (60s interval) for auto-publishing scheduled videos
  • Email: Nodemailer for password reset flows

Infrastructure

Docker Compose PostgreSQL 15 MinIO S3 Redis 7 Nginx
  • Database: PostgreSQL 15 managed via Prisma migrations
  • Object Storage: MinIO (S3-compatible) — stores videos, thumbnails, avatars, banners, watermarks
  • Cache: Redis 7 — feed caching (180s TTL), view count deduplication (30min per user), upload rate limiting
  • Proxy: Nginx reverse proxy routing /api/* to Express, media requests to MinIO

🔐 Authentication System

  • JWT Tokens: Short-lived access tokens (15min) + long-lived refresh tokens (7 days)
  • Password: bcrypt with 12 rounds salt hashing
  • OAuth: Google and GitHub OAuth2 via passport strategies
  • Password Reset: Crypto-random token sent via Nodemailer, expires in 1 hour
  • Middleware: authenticate (required auth) and optionalAuth (public routes that benefit from user context)
  • Roles: USER, MODERATOR, ADMIN — enforced at route level for admin endpoints

🎥 Video Upload & Transcoding Pipeline

Upload Flow

  1. Ingestion: Multer receives multipart form-data → temp file at server/uploads/
  2. DB Record: Prisma creates Video with status UPLOADING
  3. Raw Upload: Original file uploaded to MinIO S3 bucket
  4. Transcoding: FFmpeg background job converts to HLS (.m3u8 + .ts segments)
  5. Thumbnail: Auto-extracted at 25% duration mark (unless custom thumbnail uploaded)
  6. Classification: Videos ≤180s duration auto-classified as isShort = true
  7. Completion: Status updated to READY, SEO engines pinged

Storage Architecture

MinIO Bucket: streamsphere/
├── videos/{uuid}/
│   ├── original.mp4         # Raw uploaded file
│   ├── index.m3u8           # HLS master playlist
│   └── segment_*.ts         # HLS segments
├── thumbnails/{uuid}.jpg    # Auto or custom thumbnail
├── avatars/{uuid}.webp      # User avatar (Sharp-processed)
├── banners/{uuid}.webp      # Channel banner
└── watermarks/{uuid}.webp   # Channel watermark

▶️ Custom Video Player

A fully custom HTML5 video player built from scratch (no third-party player libraries).

  • Adaptive Streaming: HLS.js integration with auto quality detection
  • Quality Selector: Manual quality switching (144p → 4K) with HD/2K/4K badges
  • Playback Speed: 0.25x to 2x with labeled options
  • Loop Toggle: In-player loop video control
  • Captions: Toggle subtitles/closed captions
  • Theater Mode: Widescreen viewing without fullscreen
  • Picture-in-Picture: Browser native PiP via Miniplayer button
  • Keyboard Shortcuts: Space (play/pause), M (mute), F (fullscreen), ← → (seek ±10s), ↑ ↓ (volume)
  • Mobile Settings: Bottom sheet settings panel on mobile devices
  • Resume Playback: Saves and restores playback position via API every 10 seconds
  • Autoplay Next: Toggle to auto-advance to next recommended video
  • Visual Feedback: Animated overlays for play/pause/volume/seek actions

📱 Shorts (TikTok-style)

  • Classification: Any video ≤ 180 seconds = Short
  • Player: Fullscreen snap-scroll vertical feed with IntersectionObserver
  • Desktop: Centered 9:16 card with external action buttons (like, dislike, comment, share, remix)
  • Mobile: Edge-to-edge fullscreen with overlaid gradient controls
  • HLS Pooling: Preloads ±1 adjacent shorts for instant playback, destroys distant instances
  • Navigation: Arrow keys, touch scroll, and side navigation arrows
  • Comments Panel: Slide-up bottom sheet with real-time comment posting
  • Spinning Disc: Animated avatar disc that spins while playing (YouTube-style)
  • Verified Badge: Blue checkmark shown next to verified channel usernames

🔴 Live Streaming

  • Protocol: RTMP ingest (port 1935) → automatic HLS transcoding
  • Server: Node-Media-Server integrated into the Express backend
  • Studio: /studio/go-live — multi-step wizard (Details → Customization → Visibility)

✓ Channel Verification System

User Flow

  1. User goes to /settings → "Channel Verification" section
  2. Clicks "Apply for Verification" button
  3. Status changes to Pending (yellow indicator)
  4. All admin users receive a system notification
  5. Admin goes to /admin → Verification tab
  6. Admin clicks Approve or Reject
  7. User receives a notification with the outcome
  8. If approved: blue ✓ badge appears on all surfaces (channel page, watch page, shorts, video cards, home page)
  9. If rejected: user can reapply later

Badge Display Locations

  • Home page VideoCards (next to channel name)
  • Watch page (next to channel name below video)
  • Shorts player (next to @username)
  • Channel page header (next to display name)
  • Search results
  • Admin dashboard user table

⏰ Scheduled Publishing

  • Upload: Creators can set a scheduledAt datetime when uploading
  • Status: Video remains PRIVATE until the scheduled time
  • Cron Service: schedule.service.ts runs every 60 seconds
  • Auto-Publish: When scheduledAt < now(), privacy flips to PUBLIC and scheduledAt is cleared
  • Notification: Creator receives a system notification when their video goes live
